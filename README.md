BookGpt — веб-приложение для анализа текстов с использованием Yandex GPT. 
Позволяет загружать и анализировать книги, задавать вопросы по их содержанию.

Основные возможности:
📁 Загрузка текстов (ручной ввод, файл .txt, тестовая книга)

🔍 Автоматическое определение кодировки (UTF-8, Windows-1251)

✂️ Умная разбивка больших текстов на смысловые чанки

📝 Генерация краткого содержания с помощью AI

💬 Чат с ассистентом по контексту текста

🎯 Два режима работы: "Вопрос" и "Поиск"

🌐 Опциональное подключение интернета для расширенных ответов

📱 Адаптивный дизайн под все устройства

Требования
.NET 8.0 SDK или выше — Скачать с официального сайта

Пошаговая инструкция
Шаг 1: Установить .NET 8.0
Если у вас еще не установлен .NET 8.0, скачайте и установите его с официального сайта:

https://dotnet.microsoft.com/download/dotnet/8.0
После установки проверьте версию в командной строке:
dotnet --version
Должно отобразиться: 8.0.xxx или выше

Шаг 2: Скачать и распаковать проект
Скачайте архив с проектом по ссылке с Яндекс.Диска

Распакуйте архив в удобную папку (например, C:\Projects\BookGPT)

Шаг 3: Открыть командную строку и перейти в папку проекта
cd C:\Projects\BookGPT

Шаг 4: Запустить приложение
dotnet run

Шаг 5: Открыть в браузере
После успешного запуска вы увидите сообщение:
Now listening on: http://localhost:5094

Откройте браузер и перейдите по адресу:
http://localhost:5094

Остановка приложения
В командной строке нажмите Ctrl + C

📂 Как загрузить тексты книг
Способ 1: Ручной ввод
Вставьте текст прямо в текстовое поле на главной странице

Нажмите кнопку "Анализировать"

Способ 2: Загрузка файла
Нажмите на область "Выберите файл" или кликните по иконке облака

Выберите текстовый файл (поддерживаются кодировки UTF-8 и Windows-1251)

После загрузки файла автоматически появится текст в поле ввода

Нажмите "Анализировать"

Поддерживаемые форматы:

.txt — до 10 МБ

Любая кодировка (автоопределение UTF-8, Windows-1251)

Способ 3: Тестовая книга
Нажмите кнопку "Загрузить тестовую книгу" — система автоматически загрузит пример текста для демонстрации.

💡 Примеры работы с сервисом
Пример 1: Анализ тестового текста
Нажимаем на "Загрузить тестовую книгу", затем жмём "Анализировать"

Ожидаемый результат краткого содержания для тестового текста будет примерно таким:

В Москве в сером доме с белыми колоннами жила барыня-вдова.
Среди её прислуги был глухонемой дворник Герасим — мужчина богатырского телосложения, который усердно исполнял свои обязанности.
Герасим проникся симпатией к прачке Татьяне, но барыня планировала выдать её замуж за пьяницу Капитона.
Чтобы отвратить Герасима от Татьяны, её уговорили притвориться пьяной.
В итоге Татьяна вышла замуж за Капитона, а спустя год их отправили в дальнюю деревню.
Герасим спас щенка, вырастил и назвал его Муму.
Когда барыня захотела забрать Муму, собака испугалась и не подошла к ней, тогда барыня приказала убрать собаку со двора.
Герасим, после того как Муму вернулась, спрятал её, но ночью она залаяла и разбудила барыню.
В итоге Герасим утопил Муму и сбежал из Москвы в родную деревню, где продолжил работать, но после возвращения перестал общаться с женщинами и больше не держал собак.

Пример 2: Зададим вопрос.

Кто такая Муму?

Ожидаемый результат будет примерно таким:

Муму — это собака, которая живёт у Герасима.
Цитата: «Перед окном был разбит палисадник, и на самой средней клумбе, под розовым кусточком, лежала Муму и тщательно грызла кость» (ЧАСТЬ 5).

Пример 3: Поиск фрагментов по тексту.

Запрос: Степан заметил, что он её утопил.

Ожидаемый результат будет примерно таким:

Краткое описание: Степан выражает уверенность в том, что Герасим утопит собаку.
Цитата: «Ну, да, — заметил Степан, — он её утопит. Уж можно быть спокойным. Коли он что обещал...»

Пример 4: Зададим вопрос, ответ на который в тексте невозможно найти.

Кем был Печорин?

Ожидаемый результат будет таким:

В тексте нет информации об этом.
